# ComfyUI 快速入门

## 什么是 ComfyUI

ComfyUI 是一个基于节点的图形界面，用于构建 AI 图像/视频生成工作流。通过拖拽连接“节点”来创建流程，无需编程。支持 Stable Diffusion、Flux、Wan 2.2 等模型。

## 界面快速上手

- **左侧**：节点库，搜索并拖拽节点
- **中央**：画布，放置和连接节点
- **右侧**：队列管理、设置

**快捷键**：
- `Ctrl+A` 全选 | `Ctrl+C/V` 复制粘贴 | `Delete` 删除
- 鼠标中键拖拽画布 | 滚轮缩放

## 核心概念

### 节点与连接
- **节点**：功能块，如加载模型、输入文本、生成图像
- **连接**：拖拽节点间的点建立数据流
- **执行**：ComfyUI 自动按依赖顺序运行

### 工作流
一组连接的节点，可保存为 JSON 文件重复使用。

## 创建你的第一个工作流

### 文本到图像生成

1. **加载模型**：
   - 搜索 "CheckpointLoaderSimple" → 拖拽 → 选择 SD 模型

2. **输入提示**：
   - 搜索 "CLIP Text Encode" → 连接到模型的 CLIP 输出
   - 输入正向提示："a beautiful sunset over mountains"

3. **采样生成**：
   - 搜索 "KSampler" → 连接模型和文本
   - 设置：steps=20, cfg=7.0, sampler=euler

4. **解码输出**：
   - 搜索 "VAE Decode" → 连接采样器

5. **保存结果**：
   - 搜索 "Save Image" → 连接解码器

6. **运行**：
   - 点击 "Queue Prompt" → 等待生成

## 常用节点类型

- **加载**：CheckpointLoaderSimple, Load Image
- **编码**：CLIP Text Encode, CLIP Vision
- **生成**：KSampler, Latent Upscale
- **后处理**：VAE Decode, Image Scale
- **控制**：ControlNet Apply, Conditioning Combine

## 工作流管理

- **保存**：右上角 Save → 导出 JSON
- **加载**：Load → 导入 JSON 或选模板
- **模板**：右侧 Templates → 选预设（如 Video 下的 Wan 2.2）

## 解读现有工作流

拿到一个工作流（JSON 文件或界面中的），按以下步骤解读：

### 1. 加载查看
- 在 ComfyUI 点击 "Load" → 选择 JSON 文件
- 或从 Templates 加载预设
- 界面会显示所有节点和连接线

### 2. 识别核心节点
- **起点**：通常是模型加载节点（如 CheckpointLoaderSimple）
- **输入**：文本编码（CLIP Text Encode）、图像加载（Load Image）
- **处理**：采样器（KSampler）、控制节点
- **输出**：保存节点（Save Image/Video）

### 3. 理解连接关系
- 跟随连接线：数据从左到右流动
- 鼠标悬停节点查看输入/输出类型
- 注意分支：一个节点可连接多个下游

### 4. 分析关键参数
- **采样器**：steps（步数）、cfg（引导强度）、seed（随机种子）
- **分辨率**：width/height，通常在 Empty Latent Image
- **模型**：checkpoint 名称，影响风格
- **提示**：positive/negative prompts

### 5. 调试与修改
- **预览中间结果**：添加 Preview Image 节点
- **调整参数**：双击节点修改设置
- **测试运行**：Queue Prompt 查看效果
- **简化复杂流**：删除不必要节点，从基础版本开始

### 6. 学习技巧
- 从简单模板入手，逐步理解复杂工作流
- 查看节点文档：右键节点 → "View Documentation"
- 社区分享：GitHub workflow_templates 有详细说明
- 实践修改：复制工作流，尝试小幅调整

通过观察节点布局和参数，你能快速掌握工作流的逻辑和用途！

## 使用 video_wan2_2_5B_ti2v 工作流示例

以 Wan 2.2 的文本到图像到视频（Text-to-Image-to-Video）工作流为例，演示完整使用流程。这个工作流使用 5B 参数模型，从文本生成图像，再转换为视频。

### Step 1: 加载工作流
- 打开 ComfyUI，点击右侧 "Templates" → "Video" → 选择 "video_wan2_2_5B_ti2v"
- 或者下载 JSON 文件后点击 "Load" 导入
- 界面会显示完整的节点网络

### Step 2: 检查模型
- 确保已下载 Wan 2.2 5B 模型（GGUF 格式）
- 在 Unet Loader 节点选择正确的模型文件
- 如果缺失，ComfyUI 会提示自动下载

### Step 3: 设置文本提示
- 找到 CLIP Text Encode 节点
- 在 "text" 输入框输入描述："A majestic dragon soaring through stormy clouds, cinematic lighting"
- 可添加负面提示："blurry, low quality, deformed"

### Step 4: 配置视频参数
- **分辨率**：在 Empty Latent Image 节点设置 width/height（如 832x480）
- **帧数**：在 WanImageToVideo 节点设置 frames（如 16）
- **时长**：fps 设置为 8-16，根据需要调整

### Step 5: 调整生成参数
- **采样器**：steps=30, cfg=6.0, sampler=euler_ancestral
- **种子**：设置固定 seed 保证可重现，或随机
- **引导强度**：根据创意调整 cfg 值

### Step 6: 运行生成
- 点击 "Queue Prompt" 开始生成
- 观察进度条，首次运行可能需要几分钟到几十分钟
- 生成过程中可预览中间图像

### Step 7: 查看输出
- 视频保存在 ~/ComfyUI/output/video/ 目录
- 文件名通常包含时间戳和参数信息
- 在 ComfyUI 界面也可直接预览

### Step 8: 优化与迭代
- 如果结果不佳，调整提示词或参数重新生成
- 添加 ControlNet 节点控制构图
- 尝试不同分辨率和帧数平衡质量与速度

这个工作流展示了从文本到视频的完整 pipeline，适合创意视频制作！
